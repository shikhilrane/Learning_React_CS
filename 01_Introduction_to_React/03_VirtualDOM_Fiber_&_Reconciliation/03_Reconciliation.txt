************************************************************** Reconciliation **************************************************************

1. Reconciliation is a React's recurisive algorithm that reconsiders which element to update and which element not
2. Reconciliation is handled by React Fiber, the internal architecture introduced in React 16. Fiber enables incremental and interruptible rendering, allowing React to pause, prioritize, or drop updates during the reconciliation process.
3. Reconciliation is used by React to differenciate one tree with another. and what to update and what to not
4. Reconciliation is the process that uses diffing algorithm to compare new vitual dom with previous virtual dom



                        1. Fiber: Us decision ko efficiently schedule karta hai, manage karta hai, prioritize karta hai.

                        2. Reconciliation: Decide karta hai kya change karna hai (Virtual DOM ko compare karke).

                        3. React (Commit Phase): Jo changes tayaar hain, unhe browser DOM me update karta hai.
