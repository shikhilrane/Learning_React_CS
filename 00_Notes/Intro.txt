React is a front end library use to design webpages created by Meta

01. Why to learn React?
    => Hype, Job, trend, build UI
    => Makes easy to manage and build complex front end

02. When should I learn React?
    => After "Mastering JS"
    => Because most project don't need react in initial phase

03. Why react was created?
    => There was no synchronisation between state(also consider as variables) and DOM

04. Don't learn react if,
    => You don't know how JS works OR DOM works

05. React is a library
    Analogy : Homemade house is Framework (Customisation is not allowed (only fewer Customisation is allowed)) and 
              Building home from foundation is library (fully Customisation allowed)

06. Topics to learn (Foundation)
    => Core of react (state Or UI manipulation, JSX(It is like HTML within JS))
    => Component reusability (Like how to use buttons, cards etc anywhere by creating it once)
    => Reusing of Component (Props (properties are props), It is will be learnt during Component reusability like how we pass props in components)
    => How to propagate change (hooks (As we know changes doesn't get reflected to UI so to make that changes in UI we have to use hooks))

07. Additional Addon to React
    => Router (React dont have router so to use router we have to install another library called react-router)
    => State management (React doesn't have State management so we have to use another library of JS called redux, redux toolkit , zustand, context API) in return it gives hooks
    => Class Based Components (React is based on function nowadays but in initial days of react everything was class base, So we should not learn that old code but we should know how was that code)
    => BAAS Apps (Backend As A Service like Firebase, Appwrite, Superbase) these products helps us to use backend without writing backend code like social media clone or e-commerece Apps

08. After React
    => Can go to Backend
    => If we want to stay in frontend, then try to understand react as not a complete solution in most cases like, 
        1. No SEO - react doesn't come with seo means tags doesn't generate in well manner so search engines can't read that code properly
        2. Browser rendering - Code  get shipped in browser then browser generates HTMl and CSS so if browser gets this much work then our website may get slower for bigger projects
        3. No routing
    => Framework (NextJS, Gatsby, Remix) - To overcome above issue we use these Frameworks
        tip - NextJS allows us to write backend and frontend within single framework

React DOM Event Handlers
Event	Description	HTML Equivalent
onClick	Triggered when an element is clicked	onclick
onChange	Triggered when input value changes	onchange
onSubmit	Triggered when a form is submitted	onsubmit
onInput	Similar to onChange, for <input>/<textarea>	oninput
onKeyDown	Triggered when a key is pressed	onkeydown
onKeyUp	Triggered when a key is released	onkeyup
onKeyPress	(Deprecated) use onKeyDown instead	onkeypress
onFocus	Triggered when an element gains focus	onfocus
onBlur	Triggered when an element loses focus	onblur
onMouseEnter	Triggered when mouse enters an element	onmouseenter
onMouseLeave	Triggered when mouse leaves an element	onmouseleave
onMouseOver	Triggered when mouse is over an element	onmouseover
onMouseOut	Triggered when mouse leaves (similar to leave)	onmouseout
onDoubleClick	Triggered when an element is double-clicked	ondblclick
onContextMenu	Triggered on right-click	oncontextmenu
onDragStart	Triggered when a drag starts	ondragstart
onDrop	Triggered when something is dropped	ondrop
onDragOver	Triggered when dragging over a target	ondragover
onScroll	Triggered on scroll	onscroll
onLoad	Triggered when media/data loads	onload
onError	Triggered when there's an error loading media	onerror
onTouchStart	For mobile/touch devices	ontouchstart
onTouchEnd	For mobile/touch devices	ontouchend




📌 Notes:
React uses camelCase for event names: onClick instead of onclick.




You pass functions (or references), not strings:
✅ onClick={handleClick}
❌ onClick="handleClick()"




🧠 Tip: Grouped by Use Case

🖱️ Mouse Events
onClick, onDoubleClick, onContextMenu, onMouseEnter, onMouseLeave, onMouseOver, onMouseOut

🎹 Keyboard Events
onKeyDown, onKeyUp

📄 Form Events
onChange, onInput, onSubmit, onFocus, onBlur

🧲 Drag & Drop
onDragStart, onDrop, onDragOver

📜 UI Events
onScroll, onLoad, onError

📱 Touch Events
onTouchStart, onTouchEnd
